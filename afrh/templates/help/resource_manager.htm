Managing Resources
Introduction

Welcome to the Resource Manager! Viewing this page means that you have the ability to create new resources, or at least edit existing ones. On the left, you will see the Resource Description panel, which lists all available forms for adding data to this type of resource. Each form collects a different set of information, and the database aggregates this information to create a "resource". Both editing and creating are very straight-forward tasks in an Arches database, so the following tips will be enough to get you started.

Creating a New Resource

To create a new resource in the database, you only have to enter one piece of information about it. It could be any piece of information, but it's best to start with a name:

  1. From the Resource Manager menu, make sure you have chosen Historic Resource. You are now looking at an empty form, ready to record a new resource.
  2. Type "Lemee House" in the Name box, and in the type box select Primary.
  3. Click Add to add the new name and name type, and then click Save edits.
  4. Notice that at the top of the page, where it used to say "New Resource, Arches ID: n/a", it now says "Lemee House" and gives a unique identifier for the Arches ID.

Congratulations! That's all it takes to make a new resource in the database, and from here you will technically be editing it. The Value/Value Type pair (as you saw above with Name/Name Type) is a device that occurs frequently throughout the entry forms. Just be sure to always Add an attribute before you click Save edits!

Just for good measure, we'll add a Keyword to this new resource as well.

  1. Without leaving the Summary form, scroll down to the Keyword entry box.
  2. We'd like to add the Keyword living history, but we don't want to scroll all the way down the list...
  3. Begin typing "living history", and you'll see the dropdown list automatically filter keywords to match.
  4. Select living history, click Add, and click Save edits

As you can see, any dropdown lists can easily be searched by simply beginning to type the desired value.

Adding a Location

Next, let's add some geometry to our resource, so the Lemee House will show up on the map.

  1. In the panel on the left, choose the Location form.
  2. There are a few different types of location information that we can enter, but to we need to add geometry for the resource to appear on the map.
  3. In the map window, zoom navigate the map to locate the Lemee House on the corner of Jefferson St. and Demeziere St. (If you are looking for a historic resource, you can turn on any of the historic maps too!).
  4. Click Add Geometry, and choose what geometry type you'd like to draw. To make lines or polygons, just click once for each new vertex, and double-click to finish the shape.
  5. When your geometry is complete, you can select a qualifier for it if you like.
  6. Click Save edits, and then go check out the MAP tab to see your handiwork.

It's important to note that other location information can be added too--an address, a description, etc. You'll be able to search for these values like you would any other attribute, but adding geometry is the only way that you'll see the resource on the map.

Wizard Forms

Some of the input forms are designed as "wizards"--multi-step procedures designed to guide you through data collection for more complex information. Click on the Condition Assessment form to see an example of this.

  1. To begin, click +Add. You're now presented with a four-step data entry wizard.
  2. Begin by adding a condition and an assessment date. Click Add.
  3. You can now move from step to step in the wizard, without losing data (as long as you are sure to Add it).
  4. Only when you are finished recording the entire condition assessment is it time to finally click Save
Creating Resource-to-Resource Relationships

One of the most powerful features of the Arches database engine is the ability to create relationships between separate resources. For example, in the American Cemetery data present in the CRHIM, each grave is really a set of two (or more) related resources: a Historic Resource (the grave) and an Actor (the individual(s) interred in the grave). This allows for full database entries for both entities involved. The Actor, in turn, could be also related to the Historic Resource that represents his or her former home. By building these relationships, a complex web of history can be created and explored within the database.

To add relationships to a resource follow these steps:

  1. Go to the Related Resources form.
  2. If there are any existing related resources, they will be listed here. To create a new one, click +Find Resources.
  3. You are now presented a list of all the resources in the database, and three filter options to help you sift through them: the Map Filter, Attribute Filter, and Time Filter.
  4. Once you have found an existing resource by using one (or more) or these filters, click Add Relationship.
  6. You now have to choose the relationship type that best describes this new relationship. If none apply, you can select Is related to (general), or you could add a new relationship type in the RDM and go through this process again.
  7. If you like, you can also add a time period to this relationship, or a description.
  8. Once you are satisfied, click Add, and finally Save edits.

The best way to view your new relationship is to find your resource using the main SEARCH page, and click on Related Resources. You'll find more information about this in the help for the SEARCH page.

NOTE: Only one relationship may exist between any two resources, but a resource may be related to as many other resources as necessary.
